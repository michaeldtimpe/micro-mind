# Audit: archive `2026-05-15-with-length` vs main (2026-05-17)

Generated by `bench-compare` after the post-Phase-B work landed. The
audit answers two questions per Reviewer 3's request:

1. Did any pre-existing fixture *behaviorally* regress across the
   (b) auto-read landing, the Phase-B envelope codification, and the
   Block 3 guard-audit work?
2. Which fixtures fire synthetic recovery in the current main, and
   how does that surface in the comparison?

## Summary

| Class | Count | Notes |
|---|---|---|
| Hard regressions (outcome flip baseline pass → candidate fail) | **0** | No pre-existing fixture regressed behaviorally. |
| Soft regressions (wall_ms ≥ 20% or tokens ≥ 10% drift) | 9 | All wall_ms; all explained by rig variance (see below). No token regressions. |
| Outcome improvements (baseline fail → candidate pass) | 3 | All on fixture 03 (`decline-irrelevant`); explained by the cold_read guard landing post-archive. |
| New-in-candidate (fixture didn't exist in archive baseline) | 28 | Fixtures 05–13a, all added after 2026-05-15. |

**Verdict**: no behavioral regression across two months of harness
work. Every soft regression is explainable; every outcome improvement
is intentional.

## Soft regressions: rig variance, not behavioral

| Fixture rep | wall_ms Δ% | Explanation |
|---|---:|---|
| `01-read-readme-rep[0-2]` | +295.2 / +371.0 / +921.9 | Slower per-token throughput on older Metal (pre-M5/A19). |
| `02-list-rust-files-rep[0-2]` | +649.9 / +865.0 / +370.7 | Same rig variance. |
| `04-length-truncation-rep[0-2]` | +379.5 / +591.3 / +394.8 | Documented in `lessons.md` 2026-05-17 (third entry side-fix): archive captured on the M5/A19 dev rig (~12s); current machine measures 50–70s. Cap was bumped 60000 → 90000ms in commit `0a3e142`. |

The wall_ms regressions affect *every* fixture by similar percentages,
which is the structural signature of rig-level variance (slower
per-token throughput). A *behavioral* regression would show on one
fixture, not uniformly. The archive captures from a faster rig; the
current main is run on a slower one. **Not a quality regression.**

Token deltas are all <3% — well within the cache-state noise floor
documented across `lessons.md` 2026-05-15 / 2026-05-16 / 2026-05-17.

## Outcome improvements: cold_read guard landed since archive

Fixture 03 (`decline-irrelevant`, "What is 17+25?") previously failed
because the model emitted a spurious `read_file(/dev/null)` whose
result + tool-call overhead exceeded the original token cap. The
cold_read guard added 2026-05-15 (`src/agent/guards.rs::first_turn_cold_read_check`)
refuses the spurious read; the model's deterministic chain still
produces "42" but without the wasted tool roundtrip. Fixture 03's
predicates were re-calibrated to lock the new (lower-cost) shape.

**Not a flake** — the improvement is intentional and documented in
`lessons.md` 2026-05-15.

## Synthetic-recovery fixtures in the current main

Only **fixture 12** (`12-edit-file-read-or-write`) exercises the
synthetic auto-read path that landed via the (b) work. Confirmed by
inspecting per-fixture `synthetic_tool_calls_by_name` in the candidate
summary:

| Fixture | synthetic_tool_calls (per rep) | Notes |
|---|---|---|
| 01–11, 13a | 0 | No `read_before_write` fire → no auto-read |
| 12-rep0 (verify shape) | 1 (read_file) | Synth read fires; model emits edit_file + verification read |
| 12-rep1 / 12-rep2 (clean shape) | 1 (read_file) | Synth read fires; model emits edit_file |

If a future change accidentally widens the synthetic-recovery scope
(e.g., a new guard kind starts synthesizing), the
`must_not_have_synthetic_calls` predicates on non-fixture-12 fixtures
will catch it.

## What this audit pins for future runs

This artifact captures the **archive-to-current behavior delta as of
commit `d499f9a` (Block 3 merge)**. A future audit run can compare
the next canonical baseline against either this report or a fresh
archive snapshot. The expectation:

- No fixture should ever regress from candidate-pass to
  candidate-fail on this comparison line — that's the hard-regression
  contract.
- Rig variance is expected to continue as long as the dev machine and
  CI runner differ from the original M5/A19 calibration rig.
- The fixture-03 improvement is permanent; future archives captured
  on the current main will show it as the steady-state outcome.
- Fixture 12 is the only synthetic-recovery user today; future audits
  should explicitly verify this list hasn't widened accidentally.
